# الجزء الأول: فهم النظام

## ما هو تقرير.ai؟

منصة لتوليد التقارير السنوية المؤسسية تلقائياً باستخدام الذكاء الاصطناعي.

**المشكلة اللي بيحلها:**
- التقارير السنوية تأخذ أشهر لإعدادها
- جمع البيانات من 30+ جهة صعب
- كتابة التحليلات متعبة ومتكررة
- التنسيق والتصميم يأخذ وقت

**الحل:**
- نظام إلكتروني لجمع البيانات من الجهات
- AI يكتب التحليلات تلقائياً
- توليد التقرير النهائي بضغطة زر

## المفاهيم الأساسية

### 1. القالب (Template)
```
هو "الوصفة" أو "البنية" للتقرير.
مثال: "قالب التقرير السنوي لجامعة البترا"

القالب يحتوي:
├── المحاور (Axes)
├── البنود (Items)
├── الجهات المسؤولة (Entities)
└── تعريفات الجداول والرسوم
```

### 2. المحور (Axis)
```
قسم رئيسي في التقرير.
مثال:
├── المحور 1: التعليم والتعلم
├── المحور 2: البحث العلمي
├── المحور 3: خدمة المجتمع
├── المحور 4: الموارد البشرية
├── المحور 5: البنية التحتية
└── المحور 6: الشؤون المالية
```

### 3. البند (Item)
```
عنصر بيانات محدد داخل المحور.
مثال من المحور 2 (البحث العلمي):
├── البند 2.1: عدد الأبحاث المنشورة
├── البند 2.2: عدد براءات الاختراع
├── البند 2.3: التمويل البحثي
├── البند 2.4: المؤتمرات العلمية
└── البند 2.5: التعاون الدولي
```

### 4. الجهة (Entity)
```
قسم أو دائرة مسؤولة عن تقديم بيانات.
مثال:
├── كلية الهندسة
├── كلية العلوم
├── كلية الآداب
├── عمادة البحث العلمي
├── دائرة الموارد البشرية
└── الدائرة المالية
```

### 5. مكوّن البند (ItemComponent)
```
عنصر واحد داخل البند يحدد ماذا يظهر وبأي ترتيب.
مثال للبند 2.1 (الأبحاث المنشورة):

ترتيب المكونات:
1. [نص AI] → فقرة تحليلية عن الإنتاج العلمي
2. [رسم دائري] → توزيع الأبحاث حسب الكلية
3. [نص AI] → شرح للرسم
4. [جدول] → تفصيل الأبحاث حسب الكلية والتصنيف
5. [رسم أعمدة] → مقارنة مع السنوات السابقة
6. [نص AI] → استنتاجات
```

### 6. فترة الجمع (DataCollectionPeriod)
```
فترة زمنية لجمع البيانات من الجهات.
مثال: "جمع بيانات التقرير السنوي 2024/2025"

الفترة تحتوي:
├── تاريخ البداية: 1 يناير 2025
├── تاريخ النهاية: 31 يناير 2025
├── الحالة: مفتوحة / مغلقة
└── التسليمات من كل جهة
```

### 7. التسليم (Submission)
```
ما ترسله جهة معينة من بيانات.
مثال: تسليم كلية الهندسة

التسليم يحتوي:
├── الملفات المرفوعة (Excel)
├── حالة التسليم (قيد العمل / تم التسليم / معتمد)
├── نسبة الإنجاز
└── سجل المراجعات
```

### 8. المسودة (Draft)
```
المحتوى المولّد للتقرير قبل الاعتماد النهائي.

نوعان:
├── AxisDraft → مسودة محور كامل
└── ItemDraft → مسودة بند واحد
```

## المرحلة 4: تحديد مكونات كل بند ⭐ (الجديد)

### الخطوة 4.1: الدخول لصفحة المكونات
```
من صفحة /dashboard/templates/structure:

1. اختر القالب ← اختر المحور

```

### الخطوة 4.2: إضافة المكونات بالترتيب
```
مثال: البند 2.1 (الإنتاج العلمي)

اضغط "➕ إضافة مكوّن" وأضف:

┌────┬─────────────┬────────────┬─────────────────────────────┐
│ #  │ النوع       │ المصدر     │ العنوان                     │
├────┼─────────────┼────────────┼─────────────────────────────┤
│ 1  │ نص (AI) 🤖  │ توليد AI   │ مقدمة تحليلية              │
│ 2  │ جدول 📊     │ تلقائي     │ جدول (2-1) الإنتاج العلمي   │
│ 3  │ نص (AI) 🤖  │ توليد AI   │ تحليل الجدول               │
│ 4  │ رسم 📈      │ تلقائي     │ شكل (2-1) توزيع الأبحاث     │
│ 5  │ نص (AI) 🤖  │ توليد AI   │ شرح الرسم                  │
│ 6  │ رسم 📈      │ تلقائي     │ شكل (2-2) مقارنة سنوية      │
│ 7  │ نص (AI) 🤖  │ توليد AI   │ استنتاجات                  │
└────┴─────────────┴────────────┴─────────────────────────────┘
```

### أنواع المكونات المتاحة:
```
📝 نص (text)
   → نص ثابت تكتبه أنت

🤖 نص AI (text_ai)
   → نص يُولّد تلقائياً بالذكاء الاصطناعي
   → يحلل البيانات ويكتب فقرة

📊 جدول (table)
   → جدول بيانات
   → المصدر: تلقائي (من البيانات) أو يدوي

📈 رسم بياني (chart)
   → رسم دائري، أعمدة، خطي، gauge
   → المصدر: تلقائي أو يدوي

🖼️ صورة (image)
   → صورة ترفعها يدوياً

➖ فاصل (divider)
   → خط فاصل بين الأقسام
```

### مصادر المحتوى:
```
🔄 تلقائي (auto)
   → يُستخرج من البيانات المجمعة تلقائياً

🤖 توليد AI (ai)
   → يُكتب بالذكاء الاصطناعي

✍️ يدوي (manual)
   → تدخله أنت يدوياً

🔗 مرجع (reference)
   → يشير لجدول أو رسم معرّف مسبقاً
```

### الخطوة 4.3: ترتيب المكونات
```
استخدم أزرار ⬆️ ⬇️ لتغيير الترتيب.

الترتيب مهم! المكونات تظهر في التقرير النهائي
بنفس الترتيب اللي تحدده هنا.
```

## المرحلة 7: إنشاء فترة جمع

### الخطوة 7.1: الدخول لصفحة الفترات
```
URL: /dashboard/data/periods

هذه الصفحة تعرض:
├── الفترات السابقة
├── الفترة النشطة (إن وجدت)
└── زر "إنشاء فترة جديدة"
```

### الخطوة 7.2: إنشاء فترة جديدة
```
اضغط "➕ فترة جديدة"

أدخل:
├── الاسم: جمع بيانات التقرير السنوي 2024/2025
├── القالب: التقرير السنوي لجامعة البترا
├── السنة الأكاديمية: 2024/2025
├── تاريخ البداية: 2025-01-01
├── تاريخ النهاية: 2025-01-31
└── الفترة السابقة: 2023/2024 (للمقارنة)

اضغط: حفظ
```

### الخطوة 7.3: فتح الفترة
```
بعد الإنشاء، الفترة تكون بحالة "قادمة".

لتفعيلها:
1. من صفحة الفترات، اضغط "🚀 فتح الفترة"
2. أو من تفاصيل الفترة (/dashboard/data/periods/[id])

عند الفتح:
├── يُنشأ تسليم (Submission) لكل جهة
├── تصبح الحالة "مفتوحة"
└── الجهات تستطيع البدء برفع البيانات

# الجزء الرابع: توليد التقرير

## المرحلة 13: توليد المحتوى بالذكاء الاصطناعي

### الخطوة 13.1: الدخول لصفحة التوليد
```
URL: /dashboard/generate

أو: من تفاصيل الفترة → "🤖 توليد التقرير"

تعرض:
├── اختيار الفترة
├── قائمة المحاور
├── قائمة البنود (لكل محور)
├── حالة التوليد لكل بند
└── أزرار التوليد
```

### الخطوة 13.2: توليد محور كامل
```
1. اختر الفترة
2. اختر المحور
3. اضغط "توليد المحور"

النظام:
├── يجمع بيانات كل بنود المحور
├── يرسل للذكاء الاصطناعي (Claude)
├── يُنشئ AxisDraft للمحور
└── يُنشئ ItemDraft لكل بند
```

### الخطوة 13.3: توليد بند واحد
```
إذا أردت توليد بند محدد:
1. في قائمة البنود، اضغط "توليد" على البند
2. النظام يُرسل بيانات البند فقط
3. يُنشئ/يُحدّث ItemDraft
```

### كيف يعمل التوليد (خلف الكواليس)؟
```
1. قراءة مكونات البند (ItemComponent)
2. لكل مكون:
   
   إذا كان نص AI:
   ├── تجميع البيانات المتعلقة
   ├── إنشاء prompt للذكاء الاصطناعي
   ├── إرسال لـ Claude CLI
   └── حفظ النص المولّد
   
   إذا كان جدول:
   ├── استخراج البيانات من الملفات
   ├── تنسيق الجدول
   └── حفظ بيانات الجدول
   
   إذا كان رسم:
   ├── استخراج البيانات
   ├── تحديد نوع الرسم
   └── حفظ إعدادات الرسم

3. حفظ كل شي في ItemDraft.manual_content
```

## المرحلة 14: مراجعة المسودات

### الخطوة 14.1: مراجعة المسودات
```
URL: /dashboard/generate

## المرحلة 18: التصدير النهائي

### الخطوة 18.1: صفحة التصدير
```
URL: /dashboard/export

تعرض:
├── الفترات الجاهزة للتصدير
├── نسبة الإنجاز
└── خيارات التصدير
```

### الخطوة 18.2: تصدير HTML
```
اضغط "🌐 عرض HTML"
├── يفتح التقرير في تاب جديد
├── تنسيق كامل مع CSS
├── جاهز للطباعة

أو "⬇️ تحميل HTML"
├── يحمّل ملف .html
└── يمكن فتحه بدون إنترنت
```

### الخطوة 18.3: تصدير PDF
```
من صفحة HTML:
1. اضغط Ctrl+P (أو ⌘+P على Mac)
2. اختر "Save as PDF"
3. اضبط الإعدادات:
   ├── حجم الورق: A4
   ├── الهوامش: Default
   └── خلفية الرسومات: ✅
4. اضغط "Save"
```

### الخطوة 18.4: تصدير Word (قريباً)
```
⏳ هذه الميزة قيد التطوير

عند الجاهزية:
├── اضغط "📝 Word"
├── يُنشأ ملف .docx
├── قابل للتحرير في Microsoft Word
└── يحتوي كل المحتوى والجداول والرسوم
```
