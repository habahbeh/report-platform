# منصة التقارير المؤسسية — شرح شامل

> دليل كامل لفهم الفكرة، الـ Workflow، وطريقة البناء
> مع مثال عملي: التقرير السنوي لجامعة البترا 2023-2024

---

## 📌 الفكرة العامة

### ما هي المنصة؟

**منصة لبناء وإدارة التقارير المؤسسية** — تمكّن أي مؤسسة من:

1. **تصميم قالب التقرير** — تحديد المحاور والمؤشرات والجهات
2. **جمع البيانات** — كل جهة ترفع بياناتها من بوابة خاصة
3. **المقارنة السنوية** — مقارنة تلقائية مع السنة السابقة
4. **تجميع وتحليل** — النظام يجمّع ويحسب نسب التغير
5. **إنتاج التقرير** — Word/PDF جاهز مع جداول ونصوص تحليلية

### لمن؟

| القطاع | أمثلة على التقارير |
|--------|-------------------|
| **الجامعات** | التقرير السنوي، تقارير الاعتمادية |
| **الوزارات** | تقارير الإنجاز السنوية |
| **الشركات** | التقرير السنوي للمساهمين |
| **المنظمات** | تقارير المانحين |

## ⚡ المبدأ الأساسي: كل شيء ديناميكي

### لا يوجد أي كود ثابت!

المنصة **لا تحتوي على أي محاور أو مؤشرات أو جهات مكتوبة في الكود**.

كل شيء يُعرّفه المستخدم:

```
┌─────────────────────────────────────────────────────────────────┐
│                    ❌ ليس هكذا (ثابت)                           │
│                                                                 │
│   if report_type == "petra":                                   │
│       axes = ["الاعتمادية", "التدريس", "البحث"...]            │
│       items = [49 items hardcoded...]                          │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│                    ✅ بل هكذا (ديناميكي)                        │
│                                                                 │
│   Template → يُنشئه المستخدم                                   │
│      ↓                                                         │
│   Axes → يُضيفها المستخدم (عدد غير محدود)                      │
│      ↓                                                         │
│   Items → يُعرّفها المستخدم (أي عدد، أي نوع)                   │
│      ↓                                                         │
│   Entities → يُحددها المستخدم                                  │
│      ↓                                                         │
│   Entity ↔ Items → يربطها المستخدم                             │
└─────────────────────────────────────────────────────────────────┘
```

### ماذا يعني "ديناميكي"؟

| العنصر | ثابت ❌ | ديناميكي ✅ |
|--------|--------|------------|
| **المحاور** | 6 محاور مكتوبة في الكود | المستخدم يُنشئ أي عدد من المحاور |
| **المؤشرات** | 49 مؤشر ثابت | المستخدم يُعرّف أي عدد بأي اسم |
| **الجهات** | قائمة جاهزة | المستخدم يُضيف الجهات التي يريدها |
| **العلاقات** | جهة X مسؤولة عن مؤشر Y (ثابت) | المستخدم يربط أي جهة بأي مؤشرات |
| **نوع البيانات** | رقم فقط | المستخدم يختار: رقم، نسبة، نص، جدول، ملف... |
| **التجميع** | صيغة ثابتة | المستخدم يحدد: مجموع، متوسط، قائمة... |

### نفس المنصة = تقارير مختلفة تماماً

```
┌─────────────────────────────────────────────────────────────────┐
│  🏛️ جامعة البترا                                               │
│  ├── 6 محاور                                                   │
│  ├── 49 مؤشر                                                   │
│  └── 22 جهة                                                    │
├─────────────────────────────────────────────────────────────────┤
│  🏢 وزارة التربية                                              │
│  ├── 8 محاور (مختلفة تماماً)                                   │
│  ├── 120 مؤشر                                                  │
│  └── 50 جهة (مديريات)                                          │
├─────────────────────────────────────────────────────────────────┤
│  🏦 شركة مساهمة                                                │
│  ├── 4 محاور (مالي، تشغيلي، ESG، حوكمة)                       │
│  ├── 35 مؤشر                                                   │
│  └── 10 أقسام                                                  │
├─────────────────────────────────────────────────────────────────┤
│  🌍 منظمة غير ربحية                                            │
│  ├── 5 محاور                                                   │
│  ├── 25 مؤشر                                                   │

← كل هذه التقارير تُبنى على نفس المنصة
← لا يوجد كود مختلف لكل واحد
← المستخدم يُعرّف هيكل تقريره بنفسه
```

### الخلاصة

> **المنصة = محرك تقارير فارغ**
> 
> المستخدم يُعرّف:
> - ما هي المحاور؟
> - ما هي المؤشرات؟
> - ما هي الجهات؟
> - من مسؤول عن ماذا؟
> 
> المنصة تتولى:
> - جمع البيانات من كل جهة
> - التجميع والحساب
> - توليد التقرير

## 📝 شكل البيانات في التقرير

### 1. جدول المؤشرات (الجزء الأول)

كل بند يتضمن **3 قيم**:

```
┌─────────────────────────────────────────────────────────────────┐
│  البند   │  مؤشر الأداء         │ سنة   │ السنة  │ نسبة   │
│          │                      │ التقرير│ السابقة│ التغير │
├──────────┼──────────────────────┼────────┼────────┼────────┤
│   3.1    │ عدد الأبحاث في      │  187   │  142   │  31%   │
│          │ SCOPUS              │        │        │        │
└──────────┴──────────────────────┴────────┴────────┴────────┘
```

### 2. النصوص التحليلية (الجزء الثاني)

كل محور يتبعه **شرح تفصيلي** يشمل:
- تحليل الأرقام
- السياق والأسباب
- جداول تفصيلية إضافية
- رسوم بيانية (أحياناً)

**مثال من المحور 6:**
> "تحرص الجامعة على إبراز دور الطلبة في خدمة المجتمع؛ وذلك بإدماجهم في أنشطة خدمة المجتمع، والعمل التطوعي الذي يُعزّز مبادئ الإنسانية، والمواطنة الصالحة."

```
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
```

## 📊 البيانات التشاركية والتجميع

### المفهوم

بعض المؤشرات تأتي من **عدة جهات** ويجب **تجميعها**.

### مثال: عدد الأبحاث (3.1)

```
المؤشر 3.1: عدد الأبحاث المنشورة في SCOPUS و ISI

كل كلية تُدخل عدد أبحاثها:
┌─────────────────────────────┬───────────────┬───────────────┐
│ الكلية                      │ 2023-2024    │ 2022-2023     │
├─────────────────────────────┼───────────────┼───────────────┤
│ كلية الهندسة                │      28       │      22       │
│ كلية تكنولوجيا المعلومات   │      19       │      15       │
│ كلية الصيدلة                │      45       │      38       │
│ كلية العلوم الإدارية        │      31       │      24       │
│ ...                         │      ...      │      ...      │
├─────────────────────────────┼───────────────┼───────────────┤
│ إجمالي الجامعة              │     187       │     142       │
│ نسبة التغير                 │              31%              │
└─────────────────────────────┴───────────────┴───────────────┘

← كل كلية تُدخل أرقامها
← النظام يجمّع ويحسب النسبة تلقائياً
← يظهر في التقرير: الإجمالي + جدول تفصيلي
```

### مثال: أعداد الطلبة (1.8, 1.9)

```
المؤشر 1.8: عدد الطلبة غير الأردنيين

مصدر البيانات: القبول والتسجيل + الكليات
┌─────────────────────────────┬───────────────┬───────────────┐
│ الكلية                      │ 2023-2024    │ 2022-2023     │
├─────────────────────────────┼───────────────┼───────────────┤
│ كلية الهندسة                │      95       │     120       │
│ كلية تكنولوجيا المعلومات   │      78       │     105       │
│ كلية طب الأسنان             │     142       │     189       │
│ ...                         │      ...      │      ...      │
├─────────────────────────────┼───────────────┼───────────────┤
│ الإجمالي                    │     840       │    1,126      │
│ نسبة التغير                 │             -25%              │
└─────────────────────────────┴───────────────┴───────────────┘

المؤشر 1.9: نسبة الطلبة غير الأردنيين
= (840 ÷ إجمالي الطلبة) × 100 = 12.4%

← النظام يحسب النسبة تلقائياً من البيانات المُدخلة
```

### أنواع التجميع

| النوع | الاستخدام | مثال |
|-------|----------|------|
| **sum** | مجموع | إجمالي الأبحاث = مجموع أبحاث الكليات |
| **average** | متوسط | متوسط رضا الطلبة |
| **ratio** | نسبة | عدد الطلبة غير الأردنيين ÷ إجمالي الطلبة |
| **change%** | نسبة التغير | (الحالي - السابق) ÷ السابق × 100 |

## 📝 أنواع البيانات (Field Types)

### الأنواع المدعومة

| النوع | الوصف | مثال | الإدخال |
|-------|-------|------|---------|
| `number` | رقم صحيح أو عشري | عدد الأبحاث: 187 | `<input type="number">` |
| `percentage` | نسبة مئوية | رضا الطلبة: 88% | `<input>` + `%` |
| `currency` | مبلغ مالي | الإيرادات: 121,410 JOD | `<input>` + عملة |
| `text` | نص حر | ملاحظات | `<textarea>` |
| `choice` | اختيار من قائمة | التصنيف: A/B/C | `<select>` |
| `year` | سنة | سنة الاعتماد: 2024 | `<input type="number">` |
| `table` | جدول ديناميكي | توزيع حسب الكلية | جدول قابل للتعديل |
| `file` | ملف مرفق | تقرير تفصيلي.xlsx | رفع ملف |
| `calculated` | محسوب تلقائياً | نسبة التغير | لا يُدخل — يُحسب |

### مثال: المؤشر 1.9 (محسوب)

```
المؤشر 1.9: نسبة عدد الطلبة غير الأردنيين

النوع: calculated
المعادلة: (item_1.8 / total_students) * 100

┌─────────────────────────────────────────────────┐
│  المؤشر 1.8 (number): 840                       │
│  إجمالي الطلبة (من مصدر آخر): 6,774            │
│  ─────────────────────────────────────────────  │
│  المؤشر 1.9 = (840 ÷ 6774) × 100 = 12.4%       │
│                                                 │
│  ← يُحسب تلقائياً، لا أحد يُدخله               │
└─────────────────────────────────────────────────┘
```

### مثال: جدول ديناميكي (table)

```
المؤشر 3.1: عدد الأبحاث المنشورة (تفصيلي حسب الكلية)

النوع: table
الأعمدة: [الكلية, 2023-2024, 2022-2023]
الصفوف: ديناميكية (كل كلية تضيف صفها)

┌────────────────────────┬──────────┬──────────┐
│ الكلية                 │ 2023-24  │ 2022-23  │
├────────────────────────┼──────────┼──────────┤
│ كلية الهندسة           │    28    │    22    │  ← تُدخلها كلية الهندسة
│ كلية الصيدلة           │    45    │    38    │  ← تُدخلها كلية الصيدلة
│ كلية تقنية المعلومات   │    19    │    15    │  ← تُدخلها كلية IT
│ ...                    │   ...    │   ...    │
├────────────────────────┼──────────┼──────────┤
│ الإجمالي               │   187    │   142    │  ← يُحسب تلقائياً (sum)
└────────────────────────┴──────────┴──────────┘
```

## 📄 نظام توليد التقارير (Report Generation)

### المبدأ: توليد مرن ومتحكم

النظام يدعم **التوليد التدريجي** — لست مضطراً لتوليد التقرير كاملاً مرة واحدة:

```
┌─────────────────────────────────────────────────────────────────┐
│                    خيارات التوليد                               │
│                                                                 │
│   ├── توليد محور واحد                                          │
│   ├── توليد عدة محاور (multi-select)                           │

│   └── إعادة توليد محور معين فقط                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### لماذا التوليد التدريجي؟

| الميزة | الفائدة |
|--------|---------|

| **تحكم** | تعدل النص قبل التجميع |
| **عزل الأخطاء** | لو فشل محور لا يتأثر الباقي |

### مستويات التوليد

النظام يدعم التوليد على **مستويين**:


### مسودات المحاور (AxisDraft)

كل محور له **مسودة مستقلة** تمر بدورة حياة:

```
┌─────────────┐
│ not_started │  ← المحور لم يُولّد بعد
└──────┬──────┘
       │ المستخدم يضغط "توليد"
       ▼
┌─────────────┐
│ generating  │  ← AI يعمل على التوليد
└──────┬──────┘
       │ اكتمل التوليد
       ▼
┌─────────────┐
│  generated  │  ← جاهز للمراجعة
└──────┬──────┘
       │
       ├─────────────────┐
       │ تعديل يدوي      │ اعتماد مباشر
       ▼                 ▼
┌──────────┐      ┌──────────┐
│  edited  │      │ approved │
│  (معدّل) │      │ (معتمد)  │
└────┬─────┘      └──────────┘
     │ اعتماد
     ▼
┌──────────┐
│ approved │
└──────────┘

← يمكن إعادة التوليد من أي حالة (ما عدا generating)
```

### بيانات المسودة

```python
AxisDraft:
    # الربط
    period      → فترة الجمع (DataCollectionPeriod)
    axis        → المحور (Axis)
    
    # المحتوى
    content          # النص المولّد
    content_html     # HTML للعرض
    tables_data      # الجداول المولّدة
    charts_data      # الرسوم البيانية
    
    # الحالة
    status           # not_started / generating / generated / edited / approved
    version          # رقم الإصدار (يزيد مع كل تعديل)
    previous_content # المحتوى السابق (للتراجع)
    
    # Cache
    source_data      # البيانات المصدرية المستخدمة
    source_data_hash # hash للكشف عن التغييرات
    
    # AI Metadata
    ai_model         # gemini / claude / cli
    ai_tokens_input  # tokens الإدخال
    ai_tokens_output # tokens الإخراج
    ai_cost          # التكلفة بالدولار
    generation_time_ms # وقت التوليد
```

### مسودات البنود (ItemDraft)

كل بند/مؤشر له **مسودة مستقلة**:

```python
ItemDraft:
    # الربط
    period      → فترة الجمع
    item        → البند (Item)
    
    # المحتوى
    content          # النص التحليلي (2-3 جمل)
    
    # البيانات
    current_value    # القيمة الحالية
    previous_value   # القيمة السابقة
    change_percentage # نسبة التغير (محسوبة)
    
    # الحالة
    status           # not_started / generating / generated / edited / approved
    version          # رقم الإصدار
    
    # AI Metadata
    ai_model         # cli (افتراضي) / gemini / claude
    ai_tokens_output # tokens الإخراج
    generation_time_ms # وقت التوليد
```

## 🏗️ كيف تبني المشروع؟

### الخطوة 1: تحليل التقرير السابق

```
من التقرير السنوي السابق، استخرج:

1. المحاور (6 محاور في حالة البترا)
2. المؤشرات تحت كل محور (49 مؤشر)
3. نوع كل مؤشر (رقم، نسبة، مبلغ)
4. مصدر البيانات (أي جهة مسؤولة)
5. هل يحتاج تجميع من عدة جهات؟
6. الجداول التفصيلية داخل كل محور
```

### الخطوة 2: تصميم القالب في النظام

```yaml
القالب:
  الاسم: "التقرير السنوي للجامعات الأردنية"
  المرجع: "قرار مجلس التعليم العالي 3/2021"
  
  المحاور:
    - code: "1"
      name: "الاعتمادية وضمان الجودة"
      items:
        - code: "1.1"
          name: "عدد البرامج التي حصلت على الاعتماد الخاص"
          type: "number"
          unit: "برنامج"
          responsible: ["وحدة ضمان الجودة"]
          requires_previous_year: true
          
        - code: "1.8"
          name: "عدد الطلبة غير الأردنيين"
          type: "number"
          unit: "طالب"
          responsible: ["القبول والتسجيل"]
          aggregation: "sum_by_college"  # ← تجميع حسب الكلية
          
        - code: "1.9"
          name: "نسبة عدد الطلبة غير الأردنيين"
          type: "percentage"
          formula: "(1.8 / total_students) * 100"  # ← محسوب
          
  الجهات:
    - name: "وحدة ضمان الجودة"
      items: ["1.1", "1.2", "1.3", "1.4", "1.5", "1.6", "1.7", "1.10"]
      
    - name: "الدائرة المالية"
      items: ["4.1", "4.2", "4.3", ... "5.1", "5.2", "5.4"]
      
    - name: "كلية الهندسة"
      type: "college"
      items: ["3.1", "1.8"]  # ← أرقام الكلية فقط
```

### الخطوة 3: إعداد المقارنة السنوية

```
لكل فترة جمع جديدة:

1. النظام يسحب بيانات السنة السابقة تلقائياً
2. الجهة تُدخل بيانات السنة الحالية
3. النظام يحسب نسبة التغير:
   
   نسبة التغير = (القيمة الحالية - القيمة السابقة)
                  ÷ القيمة السابقة × 100
```

## 🖼️ المحتوى اليدوي (Manual Content)

### المفهوم

بعض المؤشرات تحتاج **محتوى لا يمكن توليده تلقائياً**:
- صور (شعارات، صور فعاليات، شهادات)

## 📥 استيراد بيانات السنة السابقة

### المفهوم

لحساب **نسبة التغير**، نحتاج بيانات السنة السابقة. يمكن الحصول عليها بطريقتين:

### طريقة 1: السحب التلقائي

```
إذا كانت هناك فترة سابقة مرتبطة:

DataCollectionPeriod:
    id: 2
    name: "2023-2024"
    previous_period: 1  ← مرتبطة بـ 2022-2023

عند الضغط على [سحب تلقائي]:
- النظام يقرأ current_value من ItemDrafts الفترة 1
- يحفظها كـ previous_value في ItemDrafts الفترة 2
- يحسب change_percentage تلقائياً
```

### طريقة 2: استيراد من Excel

```
1. [📥 تحميل القالب] → ملف Excel فارغ بكل البنود
2. المستخدم يملأ عمود "القيمة السابقة"
3. [📤 رفع الملف] → النظام يقرأ ويحفظ
```
